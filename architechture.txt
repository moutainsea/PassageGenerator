本项目的架构组成与功能详述如下：

一、调研当前最热的网文小说
1.直接按照当前时间查询，获取最热 top 20的小说名称、小说简介、小说分类、章节名称、小说大纲（详细）
2.可以根据给出的小说类别标签，获取最热 top 2的小说名称、小说简介、小说分类、章节名称、小说大纲（超详细）
3.将第1点获取的如上信息存为json文件，文件名称为 时间戳 + hot 20，将第2点获取的如上信息存为json文件，文件名称为 时间戳 + 小说类别标签
4.支持按平台榜单调研（起点畅销榜/番茄小说榜单/飞卢小说天榜），各平台调研产物独立存储

二、根据信息生成小说
1.根据调研出的json文件，按照如下结构生成小说：
小说名称：10字以内
小说简介：100字以内
小说分类：
小说大纲：以事件线为主，以 事件影响+事件背景+人物形象+人物行为+所处时间+所在地点 为结构（此结构在下面详述）
小说内容：以章节为单位，根据前面1个章节的标题、大纲、内容和后面1个章节的标题、大纲生成当前章节的内容

2.小说大纲的结构

以事件线为主，即 事件线分由大事件组成，大事件又由中事件组成，中事件又由小事件组成，以此类推。

每个小事件由 事件影响+事件背景+主角人物数量（大于等于1个，必选）+配角人物数量（0-3个，可选）+人物形象+人物行为+所处时间+所在地点的状态表 组成。

小事件的转移即 状态表的转移。

上述的事件线为 单独的事件库文件

下述的文件 来源于 事件库文件的内容

主角人物形象库文件：主角人物（如男一、女一、男二、女二、男三、女三等等）在各个大事件、中事件的信息，如年龄、技能、道具、性格、经历、背景等

配角人物形象库文件：记录形成的配角人物的信息（如年龄、背景、上次出现时间等），用于重复使用，但使用的情景要合理，如：故地重逢、碰巧遇到、共同目标等（继续补充）

地点库文件：记录形成的地点的信息，用于重复使用

时间库文件：记录形成的时间的信息，用于重复使用

3.分线叙事结构（最高优先级）

小说采用多分线交替叙事结构。每部小说须在 outline.subplots 中定义2条以上分线。

每条分线包含：
  - id：分线标识（如 career_line）
  - name：分线名称（如 餐饮事业线）
  - description：分线描述（该分线的主旨、涵盖内容与叙事焦点）
  - chapter_indices：属于该分线的章节序号列表
  - stages：阶段划分列表，每个阶段包含 name（阶段名）、chapters（所含章节）、summary（阶段叙述摘要）

分线交替规则：
  (1) 一个分线完成一个小阶段叙述后，切换到另一个分线叙述，避免读者疲惫。
  (2) 分线之间的切换必须保证衔接自然、前后情节合理通顺（十分重要）。
      切换处须有过渡铺垫：或以时间推移衔接、或以因果联系衔接、或以人物心理衔接，
      不得生硬跳转。
  (3) 章节标题须体现当前章节所属分线的主题特征。
  (4) 章节内容须聚焦当前分线的核心事件，同时可适当提及另一分线的进展作为呼应，
      但不得喧宾夺主。

分线设计示例（八零厨神）：
  - 餐饮事业线：主人公从小人物成长为厨神，在餐饮事业取得高成就（摆摊创业→规范化经营→工坊扩建→成名厨神）
  - 四合院生活线：主人公在四合院的生活，与邻居、亲人、朋友、情感的生活线（立足院落→亲情纽带→邻里斗争→情感收束）

4.小说内容的结构

以章节为单位，即 小说内容由一个个章节组成。
每个章节都由 章节名称+章节内容 组成。
每个章节都是对一个小事件的扩写
书写一个章节的时候 需要 根据前面1个章节的标题、大纲、内容和后面1个章节的标题、大纲生成当前章节的内容
章节须遵循分线交替规律，聚焦所属分线的核心事件

三、检查小说生成的内容、大纲、章节名称
1.在生成小说章节的内容、大纲、章节名称时，需要查看 小说总体的大纲的对应节点与之是否匹配
2.检查小说的人物、时间、地点与 人物库、时间库、地点库 的对应节点是否匹配、是否合理
3.检查小说的章节内容和章节名称是否有重复
4.检查分线交替是否合理：分线切换处是否衔接自然、章节归属是否与 subplots 定义一致

四、自动化发布流程（此点暂缓）
1.登录对应小说网站，发布小说

五、平台隔离
1.支持起点/番茄/飞卢等平台，各平台使用独立的库目录、调研文件、小说文件，互不混淆
2.各平台小说文件按平台前缀保存（如 feilu_八零厨神.json）

六、代码模块结构

models/                      # 数据模型
├── novel.py                 # 小说/章节/大纲/Subplot分线模型
├── event.py                 # 事件线（大/中/小事件+状态表）
├── character.py             # 人物（主角/配角，含技能/道具/称号）
├── location.py              # 地点库
└── timeline.py              # 时间库

storage/                     # 库管理与持久化
├── libraries.py             # 主角/配角/地点/时间库（JSON，按平台隔离）
└── research_store.py        # 调研结果存储

core/                        # 核心生成逻辑
├── llm_client.py            # LLM 客户端（多平台故障转移+mock）
├── researcher.py            # 调研模块
├── outline_builder.py       # 大纲构建（事件线+分线+回填各库+标题清洗）
├── chapter_writer.py        # 章节写作（结合前后章节+分线上下文）
├── checker.py               # 检查与反思（含分线一致性检查）
├── continuator.py           # 续写模块（加载已有小说+分线+情绪曲线参考+扩展大纲）
├── emotion_curve.py         # 情绪曲线生成器（分段情绪分析+曲线合并+检测修正建议）
├── character_consistency.py # 角色一致性检测
├── cross_platform_check.py  # 跨平台去重检查
└── generator.py             # 生成器主流程

config.py                    # 全局配置（路径/LLM/.env加载/生成规则/平台）
main.py                      # CLI 入口（research/generate/continue/generate-all/history/status）
